DSE SQL & Python 學習指南

這是一份專為 DSE 考生設計的 SQL 學習路線圖，從零基礎開始，一步一步帶你掌握 SELECT、JOIN、GROUP BY、子查詢，到 Python 中的資料處理實戰。

適合：F4–F6 學生、自學 SQL & Python 的初學者
建議使用：配合右下角 🤖 AI 學習助手，一邊做題一邊問問題

路線一：SQL 基礎到進階

對應 `practice.html` / `all-practice.html` 題庫

1
基礎查詢：SELECT + WHERE

理解如何從一張表中選出需要的欄位與紀錄。

建議：先在題庫中完成「基礎查詢」分類，再請 AI 幫你檢查寫法。

2
排序與限制：ORDER BY / LIMIT

學會控制結果顯示順序，以及只取部份資料。

可以請 AI 解釋：升序 / 降序的差異，以及常見錯誤。

3
聚合與分組：GROUP BY / HAVING

懂得使用 COUNT、SUM、AVG 等統計函數。

遇到難題時，把題目 + 你的 SQL 給 AI，請它指出問題所在。

4
多表查詢：JOIN / UNION

這是 DSE 及現實工作都非常重要的部分。

建議配合「綜合練習」中 JOIN 題目，請 AI 幫你畫出表關係。

5
子查詢與進階技巧

學會在 FROM / WHERE 中使用子查詢，提升解題彈性。

完成大部份題目後，可以請 AI 幫你總結「最常出現的 SQL 模式」。

路線二：Python 結合 SQL

對應 `python-center.html` 中的教學模塊

  • 步驟 1：熟悉 Python 基礎語法

    變數、條件判斷、迴圈、函數，建議使用 Python 中心內的範例程式。

  • 步驟 2：使用 Python 連接 SQLite

    了解 `sqlite3` 模組，學會在 Python 中執行 SELECT / INSERT / UPDATE。

  • 步驟 3：實作「小專題」

    例如：學生成績查詢系統、圖書管理、簡單報表輸出。

    可以把專題需求丟給 AI，請它幫你拆分任務與提供程式骨架。

  • 步驟 4：優化與錯誤處理

    加入例外處理、輸入驗證，並讓 AI 幫你檢查程式風格與可讀性。

善用 AI 學習助手

對應右下角 🤖 按鈕 · `ai-assistant.js`

1. 不要只叫它「給答案」

建議提示詞：「請一步一步解釋這個 SQL 怎樣想」，或「我寫錯在哪裡？」。

2. 結合題庫使用

在 `practice.html` 內，每題旁邊有「🤖 詢問 AI」按鈕，會把題目與標準答案一起交給 AI， 幫你做「檢討講解」，而不是單純出答案。

3. 做完再問，比做之前問更有效

建議先嘗試自己寫 SQL，再把「題目 + 你的答案 + 標準答案」交給 AI，請它比較差異。

4. 學習進度分析

在「學習進度」頁面中，系統會匯總你完成的題目，未來可擴充由 AI 根據你錯題紀錄，建議下一步學習重點。

常見問題 Q&A

如果還有疑問，可以直接問右下角的 AI 助手

Q1：每天應該花多少時間？

建議每天至少 20–30 分鐘，包含「看一個例子 + 做 2–3 題題目 + 反思錯題」。 比起一次做很多題，更重要的是每天有持續。

Q2：先學 SQL 還是先學 Python？

如果目標是 DSE 資料庫部分，建議優先把 SQL 基礎打穩，之後再用 Python 來做延伸應用。

Q3：遇到完全不明白的題目怎麼辦？

可以把題目完整貼給 AI，請它先用「生活例子」解釋，再回到 SQL 語法，這樣比較容易吸收。

Q4：如何知道自己準備好了？

當你在題庫中大部份分類都能正確完成，而且可以用自己的話向同學解釋題目時，就代表已經掌握得不錯。
